## 安全提示（重要）

- 本仓库默认 dev 模式仅用于本地开发与测试，不应直接用于生产。
- `.env.example` 中的默认账号/密码仅为本地开发便利，生产必须替换为独立强密钥并启用最小权限。
- 认证支持 dev token（示例：`admin` 或 `admin@space_dev`）与 hmac token（`AUTHN_MODE=hmac`）；生产必须切换到更严格的认证方式，并确保 `AUTHN_HMAC_SECRET` 等密钥不落库、不提交。
- Web 端不再内置任何默认 token：请在设置页 `/settings` 显式配置 token（可随时清除）。
- Safety/DLP 策略与出站网络治理属于生产必备项；开源仓库仅提供最小可运行示例。
- 生产环境必须配置 `API_MASTER_KEY`；服务不允许回退到 dev master key。
- 生产环境对动态 Skill（artifact 代码）默认启用信任策略：未签名/不受信的包将被拒绝执行；仅用于紧急绕过时可设置 `SKILL_RUNTIME_UNSAFE_ALLOW=true`（不推荐）。
- 动态 Skill 出站治理支持两层：
  - 兼容：`allowedDomains`（host 白名单）
  - V2：`rules`（host + pathPrefix + methods），通过 `PUT /governance/tools/:toolRef/network-policy` 下发。
- 动态 Skill 依赖扫描门槛可配置：
  - `SKILL_DEP_SCAN_MODE=deny|audit_only|off`（生产默认 deny，开发默认 audit_only）
  - `SKILL_DEP_SCAN_FAKE_JSON` 仅用于测试注入（生产不要设置）。
- 动态 Skill 运行时后端可选（默认 process）：
  - `SKILL_RUNTIME_BACKEND=process|container|auto`
  - `SKILL_RUNTIME_CONTAINER_IMAGE`（container 后端镜像）
  - `SKILL_RUNTIME_CONTAINER_FALLBACK`（container 不可用时是否允许降级到 process）。

## 本地启动

1) 启动依赖（Postgres/Redis/MinIO）

```bash
docker compose up -d
```

2) 准备环境变量

将 `.env.example` 复制为 `.env`，按需修改。

3) 安装依赖

```bash
npm install
```

4) 初始化数据库（迁移 + 种子数据 + core schema）

```bash
npm run db:seed -w @openslin/api
```

5) 启动 API

```bash
npm run dev:api
```

6) 启动 Worker（处理异步作业）

```bash
npm run dev:worker
```

7) 启动 Web/UI

```bash
npm run dev:web
```

## MVP 入口说明

- Schema/UI 等治理变更：推荐走 `governance changeset`（`POST /governance/changesets` → add items → preflight → submit → approve → release → rollback）
- Schema 发布（兼容入口，推荐用 changeset 替代）：`POST /schemas/:name/publish`
- Effective Schema：`GET /schemas/:entity/effective?schemaName=core`
- UI 页面配置：
  - `PUT /ui/pages/:name/draft`
  - `POST /ui/pages/:name/publish`、`POST /ui/pages/:name/rollback`（兼容入口，推荐用 changeset 替代）
  - `POST /ui/page-templates/generate`（从 Effective Schema 生成默认 draft）
  - `GET /ui/navigation`、`GET /ui/pages`、`GET /ui/pages/:name`
  - 个人视图偏好：
    - `GET /ui/pages/:name/view-prefs`
    - `PUT /ui/pages/:name/view-prefs`
    - `DELETE /ui/pages/:name/view-prefs`
  - Web `/p/[page]` 渲染 released 页面（list 使用 `entities.query`；detail/edit 使用 `entities.get`）
    - list 分页：使用 querystring `cursorUpdatedAt/cursorId` 透传 nextCursor
    - PageTemplate 可选 `ui` 配置块：`layout`（variant/density）、`blocks`（componentId allowlist）、列/筛选/排序/表单布局（仅影响渲染，不影响权限与数据绑定）
    - `/p/[page]` 支持 querystring 预览：`v`（variant）、`density`、`cols`（逗号分隔）
- 可插拔专业工作台（Workbench Plugin，V1）：
  - Web 宿主：`/w/[workbenchKey]`（iframe sandbox + CSP；插件通过 postMessage 请求宿主能力）

## 统一请求链路（V1）

- 认证：`Authorization: Bearer <token>`
  - dev 模式（默认）：`AUTHN_MODE=dev`，token 直接是 `subjectId[@spaceId]`（例：`admin` / `admin@space_dev`）
  - hmac 模式：`AUTHN_MODE=hmac` + `AUTHN_HMAC_SECRET`，token 为 `base64url(payloadJson).base64url(hmacSha256(secret,payloadPart))`，payload 含 `tenantId/subjectId/spaceId?/exp`
- 追踪：可选 `x-trace-id`（不传则服务端生成）；服务端为每个请求生成 `requestId`
- 回显：所有 JSON 响应都会包含 `traceId` 与 `requestId`
- 幂等：写操作使用 `idempotency-key`
- 多语言：可选 `x-user-locale` / `x-space-locale` / `x-tenant-locale` / `Accept-Language`
- 安全：`x-tenant-id` / `x-space-id` 不作为身份来源（会被忽略）

## Safety/DLP（MVP）

- 模式：`DLP_MODE=audit_only|deny`（默认 audit_only）
  - audit_only：响应/审计摘要脱敏并写 dlpSummary
  - deny：命中 token/key 类敏感信息时返回 `DLP_DENIED`
- deny 范围：`DLP_DENY_TARGETS`（默认 `model:invoke,tool:execute`）

## Memory（MVP）

- 长期记忆写入必须显式动作：`POST /memory/entries`（writePolicy=confirmed）
  - 可选：`retentionDays` 设置过期时间（MVP：1-365）
- 检索为只读入口：`POST /memory/search`（仅返回片段与引用）
- 治理：`GET /memory/entries`、`DELETE /memory/entries/:id`、`POST /memory/clear`
- 编排层工具化（MVP+）：`memory.write@1` / `memory.read@1` 通过工具执行链路触发

## Governance（MVP）

- 默认拒绝：未启用的 toolRef 不可执行（返回 `TOOL_DISABLED`）
- 启用/禁用：
  - `POST /governance/tools/:toolRef/enable`
  - `POST /governance/tools/:toolRef/disable`
- 当前稳定版本指针：
  - `POST /governance/tools/:name/active`
  - `GET /tools` / `GET /tools/:name` 返回 `activeToolRef` 与 `effectiveActiveToolRef`
- 变更集（MVP+）：draft→submitted→approved→released，支持回滚/多审批门槛
  - `POST /governance/changesets`
  - `POST /governance/changesets/:id/items`
  - `POST /governance/changesets/:id/submit|approve|release|rollback`
  - `POST /governance/changesets/:id/preflight`（只读预检摘要）
  - `POST /governance/changesets/:id/release?mode=canary`（按 canaryTargets 灰度）
  - `POST /governance/changesets/:id/promote`（提升为全量 scope）
  - `GET /governance/changesets`、`GET /governance/changesets/:id`
- 发布流水线（V2）：统一流水线摘要（gates + rollout + 回滚预览摘要）
  - `GET /governance/changesets/:id/pipeline?mode=full|canary`
- 评测准入（MVP）：高风险变更未通过评测拒绝发布（`EVAL_NOT_PASSED`）
  - `POST /governance/evals/suites`、`PUT /governance/evals/suites/:id`、`GET /governance/evals/suites`
  - `POST /governance/evals/suites/:id/runs`、`GET /governance/evals/runs`
  - `POST /governance/changesets/:id/evals/bind`、`GET /governance/changesets/:id/evals`
- 控制台页面（V2）
  - 发布流水线：`/gov/changesets`、`/gov/changesets/[id]`
  - 评测运行详情（最小）：`/gov/evals/runs/[id]`
- 渠道接入（Webhook，MVP）：验签/重放防护/幂等去重/Subject 映射/标准回执
  - `POST /channels/webhook/ingress`（入站事件）
  - `POST /governance/channels/webhook/configs`（配置：secretEnvKey、toleranceSec）
  - `POST /governance/channels/accounts`、`POST /governance/channels/chats`（映射）
- 渠道接入（Mock IM，MVP）：入站去重 + outbox 回执 + cancel
  - `POST /channels/im/mock/ingress`
  - `POST /channels/im/mock/outbox/poll`、`POST /channels/im/mock/outbox/ack`
  - `POST /channels/im/mock/cancel`
- 离线同步（MVP）：增量 push/pull + opId 幂等 + 冲突输出 + 可回放摘要
  - `POST /sync/pull`
  - `POST /sync/push`
- Knowledge/RAG（V2）：检索运营、证据链引用解析与质量评估
  - 检索日志：`GET /governance/knowledge/retrieval-logs`、`GET /governance/knowledge/retrieval-logs/:id`
  - 证据链引用解析：`POST /knowledge/evidence/resolve`（可选传 `retrievalLogId` 做引用闭环校验）、`POST /knowledge/evidence/resolveBatch`

  - 控制台页面：`/gov/sync`
