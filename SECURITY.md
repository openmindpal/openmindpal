# 安全政策（Security Policy）

## 报告安全漏洞

请不要在公开 issue 中披露安全漏洞细节。

建议使用以下方式之一私下报告：
- Email：security@openslin.dev（占位符，发布前请替换为真实邮箱或安全渠道）

报告内容建议包含：
- 影响范围与复现步骤
- 可能的利用方式与严重性判断
- 修复建议（可选）

## 响应与披露

- 我们会尽快确认收到报告，并根据严重性优先级推进修复
- 修复完成后再进行公开披露（如需），以降低被利用风险

## 安全边界提示

- 本仓库默认提供 dev 模式（例如简化 token），仅用于本地开发与测试
- 生产部署必须配置独立的密钥、DLP/Safety 策略、网络出站治理与权限策略
